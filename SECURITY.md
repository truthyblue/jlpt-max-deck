# Security Policy

## Supported version

Security fixes target the current `main` branch and the latest published public
bundle. Older snapshots are not maintained.

## Report a vulnerability privately

Please use GitHub's **Report a vulnerability** form in the repository's
Security tab. Do not open a public issue for a suspected credential leak,
publication-boundary bypass, unsafe archive handling, path traversal, or a flaw
that could overwrite files outside the selected output directory.

Include the affected commit or bundle digest, operating system, a minimal
reproduction, and the impact you observed. Do not attach copyrighted books,
generated decks, personal collections, credentials, or private filesystem data.

We will acknowledge a report as soon as practical, investigate it, and
coordinate disclosure after a fix is available. This project is maintained on a
best-effort basis and does not promise a specific response deadline.

## Public integrity concerns

Non-sensitive reproducibility failures, stale public pins, and documentation
errors can be filed through the public bug-report template.
