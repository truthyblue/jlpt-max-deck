0 · 시작 전

세 가지만 준비하면 됩니다.

처음 빌드할 때는 프로그램과 공개 자료를 내려받아야 하므로 인터넷 연결이 필요합니다. PDF는 빌드 과정에서 외부로 전송되지 않고 선택한 컴퓨터 안에서만 사용됩니다.

  • macOS 또는 Windows x64 컴퓨터
  • 공부할 기기에 설치한 Anki 앱
  • 인터넷 연결과 충분한 저장 공간
Anki가 아직 없나요?

Anki 설치 가이드에서 Windows, macOS, iPhone/iPad, Android 중 사용할 기기를 먼저 준비한 뒤 이 페이지로 돌아오세요.

1 · PDF 준비

공식 페이지에서 PDF 17개를 내려받습니다.

출판사 PDF는 공개 릴리스에 포함되지 않으며 빌더도 대신 내려받지 않습니다. 아래 공식 경로에서 지원 판본을 직접 받아 한 폴더에 모아 주세요.

출판사내려받을 자료와 공식 경로파일·쪽
해커스 N1~N5 최신 어휘 PDF와 단어장 PDF N1N2N3N4N5 10개442쪽
동양북스 일단 합격 JLPT 완벽 대비 단어장 N1~N5 N1N2N3N4N5 5개246쪽
길벗 일본어 상용한자 무작정 따라하기 1·2 핵심정리·훈련용 소책자 PDF 1권2권 2개97쪽

표가 화면보다 넓으면 표에 초점을 둔 뒤 왼쪽·오른쪽 화살표 키로 이동할 수 있습니다. 표 안의 공식 자료 링크는 Tab 키로 이동합니다.

해커스 자료는 로그인 후 N1~N5 각 교재를 보유한 상태에서 구매 도서 인증을 완료해야 내려받을 수 있습니다. 이 프로젝트는 인증에 필요한 정보를 제공하거나 인증 절차를 대신하지 않습니다. 전체 합계는 17개·지원 판본 기준 785쪽이며, 링크를 누르면 PDF 자체가 아니라 공식 자료실이나 도서 페이지가 열릴 수 있습니다.

  1. 비어 있는 새 폴더를 하나 만듭니다.

    폴더 이름과 위치는 자유롭습니다. 빌더가 재귀적으로 찾으므로 하위 폴더를 그대로 둬도 됩니다.

  2. 위 공식 페이지에서 PDF 17개를 모두 받습니다.

    해커스 10개, 동양북스 5개, 길벗 2개인지 확인한 뒤 모두 같은 전용 폴더 아래에 넣습니다.

  3. PDF를 다시 저장하거나 합치지 않습니다.

    파일명은 바꿔도 되지만 PDF 편집기에서 재저장·병합·최적화하면 검증값이 바뀌어 지원 판본으로 인식되지 않습니다.

  4. 다른 PDF가 없는지 마지막으로 확인합니다.

    지원하지 않는 PDF가 함께 있으면 잘못된 자료 사용을 막기 위해 빌드가 중단됩니다.

17개인데도 판본 오류가 나오나요?

출판사가 파일을 새 판본으로 교체했을 수 있습니다. PDF를 수정해 맞추지 말고 문제 해결 안내에서 확인할 정보와 다음 릴리스 대응 방법을 보세요.

2 · 덱 빌드

이 페이지에서 바로 빌더를 실행합니다.

Mac이나 Windows에서는 해당 탭이 자동으로 열립니다. 휴대폰에서 보고 있다면 빌드할 컴퓨터에서 이 페이지를 다시 여세요. 복사 버튼을 누르면 시작 스크립트를 안전하게 내려받아 실행하는 명령이 복사됩니다.

  1. PDF 17개를 전용 폴더 하나에 모읍니다.

    해커스 10개, 동양북스 5개, 길벗 2개입니다. 다른 PDF가 섞이지 않게 해주세요.

  2. 아래 복사 버튼을 누르고 명령을 붙여 넣습니다.

    Mac은 터미널, Windows는 PowerShell을 연 뒤 붙여 넣고 Enter를 누릅니다.

  3. PDF 폴더를 고르고 완료될 때까지 기다립니다.

    빌더가 PDF 17개와 릴리스 파일을 확인하고 덱을 만듭니다. 성공하면 public-release 결과 폴더가 자동으로 열립니다.

macOS · 터미널
시작 스크립트 보기 ↗

⌘ Command + Space를 누르고 ‘터미널’을 검색해 여세요. 복사한 명령을 붙여 넣고 Enter를 누릅니다.

(set -o pipefail; curl -fsSL 'https://github.com/truthyblue/jlpt-max-deck/raw/refs/heads/main/scripts/bootstrap-public.sh' | bash)
Windows x64 · PowerShell
시작 스크립트 보기 ↗

시작 메뉴에서 ‘PowerShell’을 검색해 여세요. 복사한 명령을 붙여 넣고 Enter를 누릅니다.

& ([scriptblock]::Create((irm 'https://github.com/truthyblue/jlpt-max-deck/raw/refs/heads/main/scripts/bootstrap-public.ps1')))

복사되는 것은 한 줄뿐입니다. 이 명령은 공개된 시작 스크립트를 내려받아 바로 실행합니다. PDF 선택부터 파일 검증과 덱 빌드까지는 시작 스크립트 안에서 처리합니다. 실행 전에 내용을 확인하려면 각 카드의 ‘시작 스크립트 보기’를 누르세요.

3 · 빌드 결과

Anki에 넣는 파일은 하나뿐입니다.

결과 폴더에는 덱과 여러 검증 리포트가 함께 생깁니다. 파일이 많아 보여도 Anki에서 선택할 것은 JLPT-MAX덱-1.0.0.apkg 하나입니다.

public-release/
├── JLPT-MAX덱-1.0.0.apkg
├── public-build-report.json
├── public-materialization-report.json
├── source-proof.json
└── 그 밖의 검증 manifest와 report
JLPT-MAX덱-1.0.0.apkg

Anki에 가져올 완성 덱입니다. 압축을 풀지 말고 파일 그대로 선택합니다.

public-build-report.json

빌드가 모든 검증을 통과했는지 기록한 요약 리포트입니다. 정상 결과는 status: passed, unresolved: 0입니다.

public-materialization-report.json

PDF에서 필요한 자료가 릴리스 계약에 맞게 복원됐는지 확인한 기술 리포트입니다. Anki에 넣지 않습니다.

source-proof.json

사용한 PDF와 source 위치의 로컬 증명입니다. PDF 관련 정보가 들어 있으므로 외부에 올리거나 문제 제보에 첨부하지 않습니다.

기억할 것: Anki에는 APKG만 가져옵니다.

APKG 파일과 source-proof.json은 개인 학습용으로만 보관하고 공유 가능한 파일·공개 링크·일반 클라우드에 올리거나 판매, 미러링, 다른 사람에게 전달하지 마세요. 아래의 같은 사용자 개인 기기 간 비공개 AnkiWeb 동기화는 별도 흐름입니다.

4 · Anki 가져오기

공부할 기기에 APKG를 직접 넣습니다.

접속한 기기의 가져오기 방법이 자동으로 열립니다. 완성된 APKG를 공부할 기기에 로컬로 복사하면 미디어까지 함께 설치되어 바로 시작할 수 있습니다.

ANKI DESKTOP

macOS에서 가져오기

  1. Anki Desktop을 실행합니다.
  2. 파일(File) → 가져오기(Import)를 누릅니다.
  3. JLPT-MAX덱-1.0.0.apkg를 선택합니다. 파일을 더블클릭해도 됩니다.
  4. 가져오기 창에서 아래의 권장 설정 5개를 확인하고 가져오기(Import)를 누릅니다.
  5. 가져오기가 끝나면 JLPT MAX덱과 음성을 확인합니다.

가져오기 창 설정

다섯 항목을 이대로 맞춥니다.

덮어쓰기(Updates)가 접혀 있으면 펼쳐서 마지막 세 항목까지 확인하세요. Anki가 이전 선택을 기억할 수 있으므로 기본값처럼 보여도 한 번씩 확인하는 편이 안전합니다.

가져오기 항목선택이유
학습 진행 상태 가져오기Import any learning progress · 복습 포함 끄기

패키지에 들어 있는 학습 일정은 가져오지 않고 모든 카드를 새 카드로 시작합니다. 다시 가져와도 내가 공부한 진도는 유지됩니다.

덱 사전 설정 가져오기Import any deck presets 켜기

JLPT MAX덱에 맞춘 새 카드 수, 카드 순서와 음성 설정을 함께 적용합니다. 끄면 Anki의 Default 프리셋이 대신 적용됩니다.

노트 타입 병합Merge note types 켜기

같은 JLPT MAX덱을 다시 가져올 때 달라진 필드와 카드 템플릿 구조를 하나로 합칩니다. 서로 다른 덱을 한 덱으로 합치는 기능은 아닙니다.

노트 업데이트Update notes If newer

가져오는 노트가 더 최신일 때만 내용을 업데이트합니다. Always나 Never로 바꾸지 않습니다.

노트 유형 업데이트Update note types If newer

카드 앞·뒷면과 스타일도 더 최신인 경우에만 갱신합니다. 한국어 화면에서도 If newer가 영어로 보일 수 있습니다.

기존 JLPT MAX덱의 필드나 카드 모양을 직접 수정했다면 먼저 백업하세요.

노트 타입 병합은 필드·템플릿 구조 충돌을 해결하며 전체 업로드 또는 다운로드 동기화를 요구할 수 있습니다. 처음 가져오는 경우에는 합칠 기존 노트 타입이 없어 그대로 새로 추가됩니다.

iPhone·iPad에서 세부 옵션이 나타나지 않으면 별도로 선택할 것이 없으므로 그대로 가져오면 됩니다. Android에서 옵션이 보이면 위와 같은 값으로 맞추세요.

각 기기에 직접 넣는 이유

APKG 안의 미디어 17,489개를 로컬에서 한 번에 풀 수 있어, 모든 음성을 별도로 내려받는 시간을 기다리지 않고 학습을 시작할 수 있습니다. 기기마다 같은 APKG 파일을 사용하세요.

기존 Anki 자료가 있는 경우

일반적인 APKG 가져오기는 기존 컬렉션을 지우지 않습니다. 다만 같은 덱의 다른 variant를 이미 사용 중이라면 노트와 템플릿이 섞일 수 있습니다. Desktop 또는 AnkiMobile은 별도 프로필에서 먼저 확인하고, AnkiDroid는 기존 컬렉션을 백업한 뒤 진행하세요.

5 · 기본 설정

처음에는 이 설정만 확인합니다.

JLPT MAX덱에는 새 카드 수와 목표 기억률을 포함한 시작용 프리셋이 들어 있습니다. APKG를 가져온 뒤 아래 경로에서 덱 옵션을 열고, 사용하는 모든 Anki 앱을 최신 버전으로 맞춘 다음 FSRS만 켜 주세요.

  3. 화면 아래쪽의 FSRS 영역으로 이동합니다.

공식 덱 옵션 안내
  1. 새 카드 수는 하루 10~20개로 시작합니다.

    가져온 프리셋은 하루 20개로 시작합니다. 매일 복습할 카드가 빠르게 늘어나므로 부담되면 지금 공부하는 덱의 New Cards/Day를 10개 이하로 낮추세요.

  2. FSRS를 켜고 목표 기억률은 90%로 둡니다.

    Desired Retention 0.90은 기억률과 복습량의 균형이 좋은 기본값입니다. 값을 높일수록 복습이 빠르게 늘어나므로 처음부터 올리지 않아도 됩니다.

  3. 기존 카드 즉시 재예약은 끈 상태로 둡니다.

    Reschedule Cards on Change를 켜면 기존 카드의 예정일이 한꺼번에 바뀌어 복습량이 급증할 수 있습니다. 끈 상태에서도 다음 복습부터 FSRS가 적용됩니다.

  4. 처음에도 Optimize를 눌러 확인할 수 있습니다.

    최신 Anki에서는 리뷰 수와 관계없이 Optimize를 사용할 수 있습니다. 기록이 적으면 현재 파라미터가 이미 최적이라는 안내가 나올 수 있으며, 이후에는 자주 누를 필요 없이 한 달에 한 번 정도 재최적화하면 충분합니다.

  5. 나머지 옵션은 기본값을 유지합니다.

    학습·재학습 단계나 최대 복습 수를 임의로 바꾸기보다 먼저 몇 주간 사용해 보고, 새 카드 수로 하루 분량만 조절하세요.

간단한 시작 · AGAIN / GOOD

처음에는 Again과 Good만 사용해도 충분합니다.

틀렸거나 기억나지 않으면 Again, 기억해 냈으면 Good을 누르세요. FSRS는 두 등급만으로도 정상 동작합니다.

네 버튼을 사용한다면 Hard는 맞혔지만 오래 망설였을 때, Easy는 거의 생각하지 않고 바로 맞혔을 때 선택하세요. 레벨이 낮거나 이미 아는 단어라는 이유만으로 Easy를 누르지는 마세요. 기억나지 않았다면 Hard가 아니라 Again을 눌러야 합니다. 잊은 카드에 Hard를 누르면 FSRS가 기억한 것으로 판단해 다음 간격이 지나치게 길어질 수 있습니다.

iPhone · iPad에서 답변 버튼 사용

질문 화면에서는 카드를 누르면 답이 열립니다. 답 화면의 기본 탭 동작은 왼쪽이 Again, 오른쪽이 Good이며, Hard와 Easy는 화면 아래의 답변 버튼을 직접 누르면 됩니다.

탭과 스와이프 동작은 설정에서 바꿀 수 있지만, 처음에는 기본 동작과 화면의 답변 버튼을 그대로 사용하는 편이 안전합니다.

Android · AnkiDroid에서 답변 버튼 사용

평가 기준에 익숙해질 때까지는 답변 버튼을 화면에 표시한 채 직접 누르는 편이 안전합니다.

6 · 첫 학습

덱 목록에서 ‘JLPT MAX덱’을 찾습니다.

가져오기가 끝나면 Desktop, AnkiMobile, AnkiDroid의 덱 목록에 JLPT MAX덱이 나타납니다. 이름 옆 펼침 버튼으로 목적별 하위 덱을 볼 수 있습니다.

어휘 음성 일상무따 종합 실전 참조표
  1. ‘JLPT MAX덱’ 또는 원하는 하위 덱을 누릅니다.

    처음이라면 어휘의 N5처럼 현재 급수에 맞는 작은 범위부터 시작해도 됩니다.

  2. ‘지금 공부하기(Study Now)’를 누릅니다.

    카드 앞면을 보고 답을 떠올린 뒤 답을 확인하고 난이도를 선택합니다.

  3. 단어와 예문 음성이 재생되는지 확인합니다.

    소리가 나지 않으면 Anki와 운영체제의 음량, 출력 장치, 음소거 상태를 먼저 확인하세요.

7 · AnkiWeb

로그인하면 학습 상태를 이어갈 수 있습니다.

무료 AnkiWeb 계정을 만들고 각 기기에서 같은 계정으로 로그인한 뒤 동기화를 실행하면 카드, 복습 기록과 현재 학습 상태가 기기 사이에서 맞춰집니다. 이미지와 음성도 함께 동기화됩니다.

AnkiWeb 무료 가입 ↗

  1. 당장 공부할 기기마다 같은 APKG를 먼저 가져옵니다.

    Mac이나 Windows에서 빌드했더라도 그 컴퓨터에 먼저 넣을 필요는 없습니다. 지금 공부할 iPhone, iPad 또는 Android로 APKG를 옮겨 가져오세요. 여러 기기를 바로 쓸 예정이라면 각 기기에 직접 가져오면 미디어 다운로드를 기다리지 않고 시작할 수 있습니다.

  2. 그 기기에서 AnkiWeb에 로그인하고 동기화합니다.

    AnkiWeb이 비어 있는 첫 동기화라면 이 기기의 컬렉션을 기준으로 업로드(Upload)를 선택합니다. 로그인만 하고 끝내지 말고 동기화 버튼도 눌러 주세요.

  3. 첫 미디어 동기화에는 시간이 걸릴 수 있습니다.

    이 덱에는 미디어 17,489개가 있습니다. APKG를 직접 가져온 기기에는 음성이 이미 있으므로 미디어 동기화 완료를 기다리지 않고 공부할 수 있습니다. 동기화를 중단했다면 나중에 다시 실행해 이어갈 수 있습니다. 미디어 동기화 상태 보는 법

  4. 다른 기기에서는 같은 계정으로 로그인해 다운로드합니다.

    처음 동기화 선택이 나오면 다운로드(Download)를 선택합니다. 이후에는 공부 전후로 동기화하면 복습 기록과 학습 상태를 이어갈 수 있습니다.

이미 AnkiWeb에 자료가 있다면 먼저 백업하세요.

첫 업로드·다운로드는 한쪽 컬렉션을 기준으로 맞추는 과정입니다. 여러 기기에서 따로 공부한 뒤 임의로 선택하면 변경이 덮어써질 수 있으므로 공식 동기화 안내를 먼저 확인하세요.

개인 동기화와 덱 공유는 다릅니다.

같은 사용자의 개인 기기 사이에서 비공개 AnkiWeb 동기화를 사용하는 것은 허용된 개인 학습 흐름입니다. APKG 파일이나 계정 접근 권한을 다른 사람에게 전달하거나 공개 링크·일반 클라우드에 올리는 것은 허용되지 않습니다.

이제 학습을 시작하세요.
