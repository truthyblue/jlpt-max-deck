# JLPT MAX Deck

> **단어를 더 깊이 익히고 실전 문제까지 다루는 JLPT Anki 덱**

JLPT MAX Deck은 한국어권 JLPT 학습자를 위한 개인용 Anki 덱 빌더입니다.
**5,800개 선별 어휘에, 근거가 확인된 경우 예문·활용·용법·단어 구성·관련어·한자
정보를 더하고 7,850개 실전 문제까지 연결했습니다.**

단어 카드 한 장을 외우고 끝나는 대신, 같은 어휘를 문장과 활용형으로 이해하고
헷갈리는 표현과 구분한 뒤 실제 문제에서 다시 확인할 수 있게 구성했습니다.

## 한 단어를 더 깊이 익히는 카드

보통의 `단어 / 읽기 / 뜻` 카드에 그치지 않습니다. 어휘에 따라 필요한 학습
정보가 답안면에 함께 붙습니다.

- 검토된 예문과 한국어 번역, 예문별 음성
- 품사와 기본 활용형
- 핵심 용법, 비슷한 표현과의 차이
- 문체·사용역
- 단어 구성과 구성 요소별 뜻
- 유의어·반의어·자동사/타동사 짝 등 관련 어휘
- 한자 정보, 사전 바로가기와 출처

모든 어휘에 모든 선택 정보가 억지로 채워지는 방식은 아닙니다. 근거가 있는
항목만 표시하고, 해당 정보가 없는 카드에서는 패널 자체가 나타나지 않습니다.

| 실제 공개판 어휘 기능 | 적용 범위 |
| --- | ---: |
| 단어 음성 | 5,800개 전체 |
| 검토된 예문 | 5,759개 어휘 · 예문 6,305개 |
| 기본 활용형 | 3,013개 어휘 |
| 관련·혼동 어휘 | 카드 표시 1,147개 어휘 · 연결 1,398개 |
| 단어 구성 | 카드 표시 199개 어휘 |
| 문체·사용역 | 104개 어휘 |
| 핵심 용법 | 74개 어휘 · 용법 76개 |
| 상위급수 한자 표기 | 101개 추가 카드 |

모든 5,800개 어휘에는 단어 음성과 음성만 듣고 떠올리는 복습 카드도 함께
제공됩니다. 낮은 급수에서 가나로 먼저 배우고 높은 급수에서 한자로 나오는
101개 어휘는 같은 노트에 상위급수 표기 카드를 하나 더 둡니다.

## `出す`의 세 용법을 익히고, 실제 문제에서 다시 확인

대표 어휘 `出す`는 `꺼내다 / 제출하다 / 편지를 보내다`의 세 의미를 각각
`かばんから財布を出す。`, `先生にレポートを出す。`,
`駅へ行く前に、この手紙を出す予定だ。`라는 검토된 예문으로 구분해 보여줍니다.
예문과 한국어 번역은 뜻별로 LLM을 활용해 만들고, 별도 LLM 판정과 규칙 검증을
거쳐 뜻·읽기·자연스러움·번역·문체에 대한 검증을 통과한 결과만 수록했습니다.
단어와 세 예문은 각각 클릭해 음성을 들을 수 있고, 기본 활용형과 관련어
`飛び出す`(뛰쳐나가다)·`送る`(보내다)도 뜻과 함께 한 답안면에 표시합니다.
여기서 끝나지 않고, 실전 덱에서는 다음과 같은 실제 용법 문제로 이어집니다.

> **‘出す’의 사용법이 올바른 문장을 고르세요.**
>
> 정답: `旅行先から家族に絵はがきを出しました。`

카드를 뒤집으면 정답 문장의 후리가나·한국어 번역·선택지별 설명과 정답 음성을
확인할 수 있습니다. JLPT 어휘 문제의 한자 읽기·표기·단어 형성·문맥 규정·
바꿔쓰기(유의 표현)·용법 6개 유형을 다루며, 출판사 교재가 각 단어를 분류한
유형과 뜻을 근거로 LLM을 활용해 그 단어가 나올 가능성이 높은 JLPT 형식의 문제를
만들었습니다. 생성안은 별도 LLM 판정과 규칙 검증을 거쳤고, 통과하지 못한 항목은
검토 후 교체했습니다.

공개판의 7,850개 실전 카드는 한자 읽기 1,758개, 표기 1,090개, 단어 형성
247개, 문맥 규정 1,624개, 바꿔쓰기 839개, 용법 2,077개와 N5 수량 표현·날짜·
월·요일 읽기 215개로 구성됩니다. 급수별 공식 유형은
[JLPT 공식 문제 구성](https://www.jlpt.jp/guideline/testsections.html)과 같습니다.

## 출처가 분명한 어휘만 골랐습니다

어휘 범위는 복수 JLPT 출판사 교재의 N1~N5 자료에서 잡았습니다. 문장형·활용형·
비표준 중복처럼 단어 카드에 부적합할 수 있는 후보를 LLM 기반으로 추가 검토해
제외했고, 의미 충돌은 별도 의미 해소 절차와 독립 판정으로 다시 확인했습니다. 같은
기반 어휘의 가나·한자 표기만 통합하고, 한자가 다른 어휘는 기본적으로 분리했습니다. LLM이 단독으로
어휘를 정하는 방식이 아니라 출판사 근거를 먼저 두고 검토를 보조하는 구조입니다.

공개판에서는 두 출판사에 모두 실린 2,281개를 `표준`, 한 출판사에 근거가 있는
3,519개를 `확장`으로 구분합니다. Anki의 `priority::02_standard`,
`priority::03_extended` 태그로 `표준`부터 시작하거나 급수별로 필터링할 수 있어,
학습 범위를 한 번에 전부 떠안을 필요가 없습니다.

출판사별로 필요한 자료는 아래 [PDF 17개 준비](#2-pdf-17개-준비)에서
확인할 수 있습니다. 이 프로젝트는 해당 출판사들의 공식 덱이나 제휴 상품이
아닙니다.

## 전체 구성

| 어휘 | 실전 문제 | 한자 | 참조표 |
| ---: | ---: | ---: | ---: |
| 5,800개 | 7,850개 | 2,337개 | 9개 |

전체 결과는 노트 15,996개, 카드 21,897개, 미디어 17,511개입니다.

완성 APKG에는 사용자의 PDF에서 복원한 출판사 파생 내용이 포함될 수 있습니다.
따라서 이 저장소는 출판사 PDF와 완성 APKG를 배포하지 않습니다. 공개 릴리스의
빌더와 데이터 bundle, 사용자가 정식으로 취득한 PDF 17개를 조합해
각자의 컴퓨터에서 개인 학습용 APKG를 만듭니다.

## 왜 완성 덱 대신 빌더인가요?

출판사 PDF의 저작권·데이터베이스권과 그 밖의 권리는 각 권리자에게 남습니다.
완성 APKG는 PDF에서 복원한 출판사 파생 필드를 포함할 수 있으므로 배포하지 않고,
사용자가 정식으로 취득한 PDF 17개로 개인 학습용 덱을 직접 만들게 합니다.

- 공개 bundle에는 출판사 PDF와 출판사 한국어 원문을 넣지 않습니다.
- 필요한 값은 사용자의 PDF text layer에서 좌표·구조 recipe로 로컬 복원합니다.
- 빌더는 필요한 PDF와 릴리스 파일이 정확히 맞는지 자동으로 확인합니다.
- 모든 검증을 통과한 뒤에만 APKG를 출력합니다.

이 방식은 DRM이나 구매 인증을 우회하지 않습니다. 필요한 자료는 각 출판사의
정식 경로로 직접 취득해야 합니다. 세부 검증 방식은 [빌드 가이드](docs/build.md)에서
확인할 수 있습니다.

## 빠른 시작

준비물은 macOS 또는 Windows x64, [uv](https://docs.astral.sh/uv/), 필요한 PDF
17개, 그리고 같은 GitHub Release의 아래 파일입니다.

현재 릴리스 후보의 17-PDF 종단 빌드는 macOS에서 확인했습니다. Windows x64
실행 경로와 플랫폼 공통 합성 테스트는 마련되어 있지만, 실제 Windows 17-PDF
수용 검증은 아직 남아 있습니다.

- `JLPT-MAX-public-bundle.zip`
- `JLPT-MAX-public-bundle.zip.sha256`
- `public-release.json`

Python 3.13과 정확한 의존성은 `uv.lock`에 따라 설치됩니다. 첫 설치에는
네트워크가 필요할 수 있습니다.

### 3. 로컬 빌드

macOS:

```bash
cd build/public-bundle
bash scripts/build-public.sh /absolute/path/to/my-jlpt-pdfs
```

Windows x64 PowerShell:

```powershell
Set-Location .\build\public-bundle
.\scripts\build-public.ps1 -PdfRoot 'C:\Users\me\Documents\jlpt-pdfs'
```

성공하면 기본적으로 bundle 옆 `public-release` 폴더에
`JLPT-MAX덱-1.0.0.apkg`와 검증 리포트가 생깁니다. 더 자세한 경로 변경법과
결과 확인법은 [빌드 가이드](docs/build.md)에 있습니다.

## 빌드가 끝나면

결과 폴더에서 Anki에 넣는 파일은
<code>JLPT-MAX덱-1.0.0.apkg</code> 하나입니다. Mac·Windows에서는 Anki
Desktop으로, iPhone·iPad에서는 AnkiMobile로, Android에서는 AnkiDroid로 이
APKG를 직접 가져올 수 있습니다. 휴대기기에 APKG를 로컬 전송해 직접 가져오면
미디어 17,511개를 별도로 내려받는 시간을 기다리지 않고 바로 학습할 수 있습니다.
설치 앱을 고르는 법, AirDrop·USB 전송과 기기별 메뉴는
[Anki 설치·가져오기 가이드](docs/anki.md)에 있습니다.

빌더는 필요한 PDF, 카드·덱 구조와 미디어가 해당 릴리스와 맞는지 자동으로 확인합니다.
모든 검증을 통과한 경우에만 APKG와 리포트를 출력합니다. 세부 검증값과 결과
확인법은 [빌드 가이드](docs/build.md#4-성공-결과-확인)에 있습니다.

## 공개 경계

이 저장소와 릴리스에는 서로 다른 권리 경계를 가진 세 층이 있습니다.

1. **빌더 코드** — `AGPL-3.0-or-later`. 수정한 빌더 코드는 라이선스에 따라
   공유할 수 있습니다.
2. **공개 bundle** — KANJIDIC2처럼 자체 오픈 라이선스가 있는 데이터와,
   개인용 로컬 빌드를 위해 제공되는 프로젝트 recipe·생성 콘텐츠·음성이
   함께 있습니다. 코드 라이선스가 bundle 전체에 자동 적용되는 것은 아닙니다.
3. **사용자가 만든 APKG** — 사용자 PDF에서 로컬 복원한 출판사 파생 필드를
   포함하므로 개인 학습용입니다. APKG 파일을 업로드, 미러링, 판매, 공유하지
   마십시오. 같은 사용자의 개인 기기 사이에서 비공개 AnkiWeb 동기화를 사용하는
   것은 별도의 개인 학습 흐름입니다.

공개 릴리스에는 완성 APKG, 출판사 PDF, PDF 추출물 또는 `source-proof.json`을
넣지 않습니다. 자세한 조항은 [NOTICE](NOTICE)와
[저작권·라이선스 설명](docs/privacy-and-licensing.md)을 확인하세요.

## 안전장치

- 필요한 PDF 17개가 정확히 일치하지 않으면 즉시 중단
- 빌더와 데이터 bundle이 같은 릴리스인지 검증
- 필요한 값과 최종 덱 구조가 릴리스 기준과 맞는지 검증
- 임시 폴더에서 완성·검증 후에만 결과 폴더를 교체
- 기존의 검증된 결과가 있으면 실패 시 그대로 보존

세부 검증값과 공개 검증 계약은 [빌드 가이드](docs/build.md)에서 확인할 수 있습니다.

## 지원과 기여

오류를 제보할 때 PDF나 추출 문자열을 첨부하지 마세요. 운영체제, 실행 명령,
오류 메시지, `public-build-report.json`의 비민감 항목만으로 재현 정보를
구성해 주세요. 빌더 개선은 환영하지만 publisher 자료나 완성 덱을 PR에 넣으면
안 됩니다.

이 프로젝트는 Anki, 해커스, 동양북스, 길벗 또는 EDRDG의 공식 제품이 아니며,
어떤 출판사의 후원이나 보증도 받지 않습니다.
